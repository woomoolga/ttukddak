# 뚝딱 (TTUKDDAK) 설계 문서

> 작성일: 2026-06-25
> 상태: 확정

## 1. 개요

**뚝딱**은 사업자번호 하나로 받을 수 있는 모든 혜택을 정확하게 안내하고,
사업에 필요한 IT 서비스를 원스톱으로 제공하는 플랫폼이다.

- **도메인**: ttukddak.woomoolga.com
- **계정**: admin@woomoolga.com
- **프로젝트 경로**: ~/Documents/side-projects/ttukddak

## 2. 비즈니스 구조

```
혜택 조회 (무료) → 사업자 유입 → IT 원스톱 서비스 (유료)
                 ↓
            이메일 확보 → 마케팅 채널
                 ↓
            회원가입 유도 → 재방문 + 알림
```

### 수익 모델

- **1차**: 광고 수익 (세무사, 법무사, 사업자 대출 등 고단가 광고)
- **2차**: IT 서비스 수주 (직접 제공)

## 3. 타겟 사용자

| 유형 | 설명 |
|------|------|
| 소상공인/자영업자 | 음식점, 카페, 미용실 등 |
| 스타트업/중소기업 | 법인 사업자 |
| 예비 창업자 | 창업 준비 중 |
| 간이과세자 | 소규모 개인사업자 |

- **서비스 지역**: 전국 (온라인)
- **오프라인 서비스**: 대전/충남 지역만 (네트워크 공사, 컴퓨터 수리 등)

## 4. 핵심 기능

### 4-1. 사업자번호 조회 + 혜택 목록 (메인)

```
사업자번호 입력
    ↓
자동 파악: 사업자 유형(개인/법인/간이) / 업종 / 지역
    ↓
결과 화면:
  ├─ 받을 수 있는 혜택 목록 (마감일순 정렬)
  │   ├─ 혜택명
  │   ├─ 지원 금액/내용
  │   ├─ 신청 자격 요건
  │   ├─ 마감일
  │   └─ 신청 방법 + 링크
  └─ 이 업종에 맞는 마케팅 팁 (AI 생성)
```

**핵심 가치**: 정확한 정보 전달. 잘못된 혜택 안내는 신뢰를 무너뜨린다.

### 4-2. 이메일 결과 전송

- 비회원도 조회 결과를 이메일로 받기 가능
- 이메일 확보 → 마케팅 채널 확보
- 새 혜택 알림 등 후속 마케팅 가능

### 4-3. 소셜 로그인 + 저장

- **로그인 방식**: 카카오 / 네이버 / 구글
- **회원 기능**: 조회 결과 저장, 새 혜택 알림
- **비회원**: 조회 + 이메일 전송만 가능

### 4-4. 업종별 AI 마케팅 가이드

- 사업자번호에서 추출한 업종/지역/규모 기반
- Gemini AI가 맞춤형 마케팅 전략 생성
- 업종이 수백 가지이므로 정적 콘텐츠가 아닌 AI 생성 방식

### 4-5. IT 서비스 견적 요청

- 서비스 목록 + 견적 요청 폼
- 혜택 조회로 유입된 사업자가 자연스럽게 서비스 발견

## 6. 데이터 소스 및 업데이트

### API 실시간 조회
- 국세청 사업자 조회 API (사업자 유형/상태)

- AI(Gemini)가 공고 내용 파싱 → 대상, 조건, 마감일, 신청 방법 구조화

## 11. 스케일링 시 예상 문제점 및 대응

### API 호출 제한 (위험도: 높음)

| 데이터 소스 | 제한 | 대응 |
|------------|------|------|
| 공공데이터포털 | 운영계정 일 10만 건 | 조회 결과 DB 캐싱, 동일 사업자번호 재조회 시 캐시 반환 |
| 국세청 사업자 조회 | 일 호출 제한 | 캐싱 + 배치 조회 |
| 기업마당 | 별도 확인 필요 | API 키 다중 운영 |

### 데이터 정확성 클레임 (위험도: 높음)

- 마감된 혜택이 여전히 표시되면 신뢰 하락
- **대응**: 면책 문구 필수 ("본 정보는 참고용이며, 정확한 내용은 해당 기관에 확인하세요")
- 주기적 데이터 업데이트 + 마감일 자동 비활성화

### 비용 증가 (위험도: 중간)

| 항목 | 무료 범위 | 초과 시 |
|------|----------|--------|
| Vercel | 월 100GB 대역폭 | 유료 전환 |
| Supabase | 500MB DB, 5만 MAU | 유료 전환 |
| Gemini API | 무료 티어 | 키 추가 또는 유료 |
| Resend | 월 3,000건 이메일 | 유료 전환 |

### 개인정보 이슈 (위험도: 중간)

- 이메일 수집 시 개인정보보호법 적용
- 마케팅 이메일 발송 시 수신 동의 필요
- **대응**: 이메일 수집 시 마케팅 수신 동의 체크박스, 개인정보처리방침 페이지 필수

### 크롤링 법적 리스크 (위험도: 낮음)

- 공개된 정보 크롤링은 원칙적으로 합법
- 지자체 이용약관 위반, 과도한 서버 부하 유발 시 문제 가능
- **대응**: robots.txt 준수, 크롤링 간격 조절, 공식 API 우선 사용
